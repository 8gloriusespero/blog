Building a useful Macintosh application with Nuklear and CoprocessorJS (Messages for Macintosh)

Finally after building up lots of tooling for classic Macintosh, we can develop a full application.

If you’re stumbling on this article without context, you might want to start with these to get up to speed:

  • A simplified guide to classic Macintosh development using Retro68 and pce-macplus
  • Introducing CoprocessorJS
  • Introducing Nuklear for Macintosh

The motivation

Ultimately I want to use a Macintosh Classic all day, every day, for useful things and have it be a major part of my workflow.

To that end, I want to create an iMessage client to send and receive messages with my friends, and I want to run it on System 6.

The major components

  • The Macintosh Classic and the software that we’ll be running on it - the “old” Macintosh app
  • An intermediary “modern” computer connected to a serial port on the Macintosh via an RS422 to USB cable (could be a Raspberry Pi)
  • A CoprocessorJS instance running on the “modern” computer connected to the Mac’s serial port - the “glue” app
  • A Mac running a small nodejs server application, signed in to an iMessage account - the “new” Mac app
